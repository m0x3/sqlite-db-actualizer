# SQLite DB Actualizer

Скрипт на PHP 7.3 для синхронизации SQLite-базы данных с эталонным SQL-файлом. При обновлении приложения приводит существующую БД в соответствие с новым эталоном без потери данных.

## Требования

- PHP 7.3+ с расширением `pdo_sqlite`

## Использование

```bash
php db_actualizer.php --db=<путь к БД> --sql=<путь к SQL> [--dry-run] [--verbose]
```

### Параметры

| Параметр | Описание |
|---|---|
| `--db` | Путь к файлу SQLite базы данных |
| `--sql` | Путь к эталонному SQL-файлу |
| `--dry-run` | Только проверка, без внесения изменений |
| `--verbose` | Подробный вывод (показывает выполняемые SQL-запросы) |

### Примеры

```bash
# Проверить что изменится, не трогая БД
php db_actualizer.php --db=/var/app/data.sqlite --sql=/var/app/db.sql --dry-run

# Применить изменения с подробным выводом
php db_actualizer.php --db=/var/app/data.sqlite --sql=/var/app/db.sql --verbose

# Тихое применение (только итог)
php db_actualizer.php --db=/var/app/data.sqlite --sql=/var/app/db.sql
```

### Коды возврата

| Код | Значение |
|---|---|
| `0` | Успех (изменений нет или все применены без ошибок) |
| `1` | Произошли ошибки при применении изменений |

## Что скрипт синхронизирует

### Таблицы
- Создаёт таблицы, которые есть в эталоне, но отсутствуют в БД
- Удаляет таблицы, которых нет в эталоне

### Столбцы
- Добавляет отсутствующие столбцы (`ALTER TABLE ... ADD COLUMN`)
- Удаляет лишние столбцы — через пересоздание таблицы с сохранением данных
- Исправляет тип данных, значение DEFAULT, флаг NOT NULL — через пересоздание

> При пересоздании таблицы данные полностью сохраняются: копируются все столбцы, которые есть и в старой, и в новой версии таблицы.

### Индексы
- Создаёт отсутствующие индексы (обычные и UNIQUE)
- Удаляет лишние индексы
- Пересоздаёт индексы, у которых изменился список столбцов или флаг UNIQUE

### Views и Triggers
- Создаёт отсутствующие
- Удаляет лишние
- Пересоздаёт изменившиеся (DROP + CREATE)

### Начальные данные
- При создании новой таблицы выполняет `INSERT`-операторы из эталонного файла, если они там есть

## Пример вывода

```

Изменения (5):
  + ADD COLUMN `bonus_pay` to `partners`
  + CREATE TABLE `device_synchronization`
  + CREATE INDEX `serial_number` ON `device_synchronization`
  + DROP INDEX `old_idx` (лишний)
  + DROP TABLE `legacy_table`
```

```

  + [DRY-RUN] ADD COLUMN `score` to `users`
  + [DRY-RUN] CREATE INDEX `users_score` ON `users`
```

```

Изменений не требуется. БД соответствует эталону.
```

## Тесты

```bash
php7.3 tests/DbActualizerTest.php
```

Тесты охватывают 46 сценариев, сгруппированных по разделам:

- **Парсинг SQL** — разбор `CREATE TABLE`, `CREATE INDEX`, `DEFAULT`, типов, `BEGIN...END` в триггерах
- **Создание таблиц** — добавление отсутствующих, идемпотентность при полном совпадении
- **Добавление столбцов** — одного, нескольких, с DEFAULT
- **Удаление столбцов** — через пересоздание таблицы, сохранение данных
- **Изменение типа/DEFAULT** — пересоздание с сохранением данных
- **Удаление таблиц** — удаление лишних
- **Индексы** — создание, удаление, пересоздание при изменении
- **Dry-run** — не вносит изменений, корректно сообщает о различиях
- **Views** — создание, удаление
- **Triggers** — создание, удаление
- **Комплексные сценарии** — несколько изменений одновременно, реальный `db.sql`
- **Граничные случаи** — `AUTOINCREMENT`, составные индексы, пустой эталон, кавычки в именах

## Использование в PHP-коде

```php
require 'db_actualizer.php';

$actualizer = new DbActualizer(
    '/path/to/data.sqlite',  // путь к БД
    '/path/to/db.sql',       // путь к эталону
    false,                   // dry-run: false = применять изменения
    true                     // verbose: выводить подробности
);

$ok = $actualizer->run();

if (!$ok) {
    foreach ($actualizer->getErrors() as $error) {
        error_log($error);
    }
}

// Список применённых изменений
$changes = $actualizer->getChanges();
```

## Ограничения SQLite

SQLite не поддерживает `ALTER TABLE` для изменения типа столбца или удаления столбца. Скрипт обходит это через стандартный приём: создаёт временную таблицу по эталону, копирует данные, удаляет старую таблицу, переименовывает временную. Операция выполняется в транзакции — при ошибке откатывается.

Столбцы, которые есть в старой таблице, но отсутствуют в новой, при копировании отбрасываются. Новые столбцы получают значение NULL или DEFAULT.
